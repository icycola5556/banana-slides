🎨 CoverLayout 颜色一致性修复演示

8 个测试全部通过

场景 1: 有背景图时的颜色

已修复

当 background_image 存在时，文字应使用深色以确保在浅色背景上可读。

修复前 (Bug):
color: '#ffffff' // 强制白色

修复后:
color: background_image ? '#111111' : '#ffffff'

场景 2: 无背景图时的颜色

已修复

当 background_image 不存在时（使用渐变背景），文字保持白色。

修复前 (Bug):
color: '#ffffff' // 虽然正确但无判断

修复后:
color: background_image ? '#111111' : '#ffffff'

📋 修复总结

  • 问题描述：预览 PPT 界面中，有背景图时文字显示为黑色（正确），但导出 HTML 时文字被强制设为白色（错误），导致文字在浅色背景上看不见。
  • 根本原因：React 组件预览和 renderCoverLayoutHTML 导出函数中的颜色逻辑不一致。
  • 修复文件：
    • frontend/src/experimental/html-renderer/layouts/CoverLayout.tsx
    • frontend/src/experimental/html-renderer/layouts/common/CoverLayout.tsx
  • 修复内容：
    • 标题颜色: background_image ? '#111111' : '#ffffff'
    • 副标题颜色: background_image ? '#333333' : 'rgba(255,255,255,0.9)'
    • 作者颜色: background_image ? '#333333' : 'rgba(255,255,255,0.8)'
    • 文字阴影也同步修复，确保与预览一致
  • 测试覆盖：
    • 8 个单元测试验证颜色逻辑正确性
    • 覆盖有背景图/无背景图两种场景
    • 验证 text-shadow 阴影效果一致性
    • 验证 Variant B 布局变体
